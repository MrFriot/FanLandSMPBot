# FanLandSMPBot

Телеграм-бот на aiogram 3: следит за Minecraft-сервером и присылает
уведомления о его запуске и остановке.

## Запуск

1. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN`, `MC_HOST`, `MC_PORT`.
2. `docker compose up -d --build`

## Команды

- `/start` — подписаться на уведомления
- `/track <ник>` — уведомления о входе/выходе игрока (в этом чате/теме; регистр ника важен)
- `/untrack <ник>` — перестать отслеживать
- `/seen <ник>` — профиль игрока: последние сессии, привычки, доля онлайна
- `/top [дней]` — топ игроков по времени в игре (по умолчанию за 7 дней)
- `/stats [дней]` — дашборд сервера картинкой: сводка, топ-5, графики по дням и часам (при сбое рендера — текстом)
- `/stop` — отписаться от уведомлений о сервере
- `/status` — текущее состояние сервера (игроки, пинг, версия)

## Структура

- `bot/config.py` — настройки из `.env`
- `bot/storage.py` — хранилище подписок (SQLite; интерфейс позволяет заменить на Redis/Postgres)
- `bot/monitor.py` — фоновый опрос сервера, рассылка уведомлений, запись сессий
- `bot/sessions.py` — история игровых сессий (вход/выход всех игроков)
- `bot/handlers/` — команды; новые фичи добавляются отдельными роутерами
- `bot/main.py` — точка входа

## Тесты

Юнит-тесты не ходят в сеть (Telegram и Minecraft-сервер подменяются
фейками) и не требуют зависимостей сверх requirements.txt:

```
python3 -m unittest
```

## История сессий

Бот пишет в ту же SQLite-базу вход и выход каждого игрока — независимо
от подписок в Telegram (таблица `sessions`, одна строка — одна сессия;
`left_at IS NULL` значит «игрок сейчас онлайн»). Время — unix-секунды (UTC).

Точность ограничена периодом опроса (`CHECK_INTERVAL`). Пока бот
выключен, история не пишется: при старте открытые сессии сверяются
с реальным составом сервера, расхождения закрываются временем сверки.

Все даты и время в сообщениях бота — по МСК (UTC+3).
